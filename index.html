Pros

  • super easy to implement
  • pages always look the same for a given breakpoint/screen target, no matter the screen width
  • <br> tags are actually useful!
  • great results with only 2 breakpoints (and consequently, only 2 designs)
  • just as easy for 3 or more targets (eg. mobile, tablet, desktop)

Cons

  • rem units only, so be prepared to override pixel values everywhere
  • 3rd party plug-ins with pixels can be a nightmare
  • font-size scaling in the browser doesn't really work
  • does not work that well for apps (since resizing the window to get more screen estate has no effect)
  • desktop view can feel too large with only 2 breakpoints
